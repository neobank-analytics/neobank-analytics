  • Montant anormal — z-score vs moyenne du user
  • Pays nouveau — premier paiement dans ce pays (chronologique)

Pas de label fraude → validation par proxy REVERTED. Les suspectes ont ×2 plus de REVERTED que la moyenne. Le modèle capte des anomalies réelles.

Conclusion — 3 leviers actionnables

Recommandations business

🟢 Engagés (19%)
Programme ambassadeur. Accès anticipé aux nouvelles features. Ils sont déjà convaincus — transformez-les en promoteurs de la neobank.
🟡 Réguliers (59%)
Activation crypto — le marqueur d'engagement. Upsell vers Premium. Notifications PUSH personnalisées. C'est le plus gros réservoir de croissance.
🔴 À Risque (22%)
Campagne de réengagement urgente via PUSH. Offre incitative (cashback, premium gratuit 1 mois). Accepter l'attrition résiduelle — concentrer le budget.
→ le message
Le crypto n'est pas qu'une feature — c'est un levier d'engagement. Le SMS est mort, le PUSH est roi, et 22% de la base nécessite une intervention immédiate.
Conclusion — Rigueur

Limites & ouvertures

Limites assumées
Churn à 90 jours — seuil fixe, identique pour tous les segments. Un Engagé inactif 90j ≠ un Régulier inactif 90j.
Pas de label fraude — proxy REVERTED (×2 enrichissement) encourageant mais insuffisant pour la production.
Crypto drive le clustering — feature binaire qui crée une coupure nette après StandardScaler. Les Réguliers à 0% crypto est un artefact structurel.
Data leakage potentiel — fréquence calculée sur toute la période, y compris la phase d'inactivité.
Pistes d'amélioration
Seuil par segment — 60j pour les Engagés, 120j pour les Réguliers. Plus adapté au comportement réel.
A/B test notifications — on a ×1.9 corrélation mais pas de causalité. Test randomisé nécessaire.
Clustering sans crypto — retirer le crypto des features et l'analyser en post-hoc. Segments basés sur le comportement pur.
Score fraude temps réel — pipeline de scoring à la transaction avec seuil d'alerte configurable.
